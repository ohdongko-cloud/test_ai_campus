# PRD: 푸터 정책 모달 활성화 + 관리자 모드 버튼 위치 이동

- 작성일: 2026-05-26
- 작성자: Claude (요청자: <오너>)
- 범위: 푸터 링크 4개 활성화 / 정책 모달 3종 신규 / 상단 관리자 버튼 → 푸터 텍스트 버튼으로 이동

---

## 1. 배경 / 문제

### A. 상단 관리자 버튼이 시각적 노이즈
홈 헤더 우측에 노출된 "관리자" 버튼이 일반 사용자에게도 보임 → 사이트 전반의 톤이 어수선. 1,800명 전 직원 오픈을 앞두고 일반 사용자에게 필요 없는 입구가 가장 잘 보이는 위치에 있음.

### B. 푸터 정책 링크가 모두 비활성
[개인정보처리방침 | 이용약관 | 관리자 문의] 3개 텍스트가 노출만 되고 클릭해도 동작 안 함 → 신뢰도/완성도 저하.

특히 영상의 외부 유출 방어를 강화한 직전 작업과 맞물려, **이용약관에서 외부 유출 시 인사 조치 문구**가 필요. 워터마크 + 약관 명문화 = 사회적 억제력 극대화.

## 2. 정책 결정 (사용자 확정)

| 항목 | 결정 |
|---|---|
| 운영 주체 표기 | **이랜드리테일 AX팀** / 대표 이메일 `<관리자-이메일>` |
| 관리자 문의 동작 | `https://open.kakao.com/o/ssiKWcTf` 새 탭 열기 (카카오 1:1) — 모달 X |
| 표시 방식 | **모달 팝업** (페이지 내 읽고 닫기) |
| 약관 강도 | **명확한 인사조치·소송 가능성 포함** (영업비밀보호법·저작권법 등) |

## 3. 목표 / 비목표

### 목표
- G1. 상단 헤더의 **"관리자" 버튼 제거** (PC + 모바일 드로어 모두).
- G2. 푸터 끝에 **"관리자 모드" 텍스트 버튼 추가** (개인정보 / 이용약관 / 관리자 문의 옆).
- G3. **개인정보처리방침 모달** — 클릭 시 노출, 신뢰도 확보 수준의 내용.
- G4. **이용약관 모달** — 외부 유출 시 인사조치·법적 책임 명문화.
- G5. **관리자 문의** 클릭 시 카톡 1:1 새 탭 (모달 X).

### 비목표
- 정책 문서 별도 페이지 URL (`/privacy`, `/terms`).
- 다국어 번역.
- 정책 변경 이력 자동 알림.
- 관리자 페이지에서 정책 텍스트 직접 편집.

## 4. 기능 요구사항

### F1. 관리자 버튼 위치 이동
- `app/page.tsx`:
  - 상단 헤더의 PC `<button>관리자</button>` 블록 제거.
  - 모바일 드로어의 `관리자 모드` 항목도 제거.
  - 푸터에 **텍스트 링크** 형태로 추가: "관리자 모드".
  - 클릭 동작은 기존과 동일 (권한 회원이면 즉시 진입, 비권한자는 비번 모달).

### F2. 푸터 링크 활성화
- 현재 `['개인정보처리방침', '이용약관', '관리자 문의'].map(...)` 정적 렌더링 → 핸들러 부여.
- 정책 2종은 모달 오픈, 관리자 문의는 `window.open` 외부 링크.

### F3. 정책 모달 컴포넌트
- 신규 `components/LegalModal.tsx` 재사용 가능한 모달 (제목·본문·닫기).
- 신규 `components/policy/PrivacyContent.tsx` — 개인정보처리방침 본문 (TSX 문서).
- 신규 `components/policy/TermsContent.tsx` — 이용약관 본문.
- 모달 표시 상태는 `app/page.tsx` 의 `'privacy' | 'terms' | null` state로 관리.

   - 부정 사용·외부 유출 방지

7. **쿠키·세션**: 로그인 유지(httpOnly), 어드민 세션 — 보안 정책에 따라 사용

9. **시행 시기 및 변경**: 시행일 명시, 향후 변경 시 사전 공지

## 7. 엣지 케이스

| 케이스 | 동작 |
|---|---|
| 모달이 화면보다 긴 경우 | 내부 스크롤 (`max-height: 85vh; overflow-y: auto`). |
| 모달 열린 상태에서 ESC | 닫힘 (keydown 리스너). |
| 푸터에서 관리자 모드 클릭 | 기존 비번 모달 또는 즉시 진입 (권한 보유 시) — 기존 동작 유지. |

| 관리자 문의 카톡 링크 만료/변경 | 운영자가 코드 수정 + 배포 (DB 동적 관리 안 함). |
