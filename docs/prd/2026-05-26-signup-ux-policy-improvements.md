# PRD: 회원가입 UX·정책 개선

- 작성일: 2026-05-26
- 작성자: Claude (요청자: <오너>)
- 범위: WelcomePopup + 비번 정책 + 가입 직후 분기

---

## 1. 배경 / 문제

기존 회원가입 흐름에서 운영 피드백:
1. "부서(팀)" 라벨이 모호 — 이랜드 그룹사 특성상 브랜드(예: 뉴발란스, 후아유 등) 단위 소속도 흔하므로 명확히 표기 필요.
2. 직무 플레이스홀더 "사원/대리/과장"이 직급에 가까운 표현 → 실제 직무(상품기획·생산·점장·인사 등)와 거리.
3. 간편 비번이 4~12자 영숫자만 허용 — 짧고 단순한 비번으로 모든 사용자가 통일될 위험. 1,800명 모수에선 무차별 대입 우려.
4. 회원가입 완료 시 자동으로 홈으로 이동 — 가입 직후 "무엇부터 해야 할지" 막막. 컨버전·온보딩 개선 필요.

## 2. 목표 / 비목표

### 목표
- G1. **라벨 변경**: "부서(팀)" → **"부서(브랜드/팀)"**
- G2. **직무 플레이스홀더 변경**: "예: 사원 / 대리 / 과장" → **"예: 상품기획, 생산, OO OO점 점장, 지점 인사팀"**
- G3. **비번 정책 강화**: 4~12자에서 **8~16자**로 길이 확장 + **영문 + 숫자 + 특수문자** 모두 1개 이상 포함 필수.
- G4. **가입 완료 후 분기**: 가입 성공 시 환영 화면에서 "AI 학습 시작하기" / "홈에서 둘러보기" 두 버튼 제시.

### 비목표
- 로그인 폼의 비번 정책 변경 (기존 회원은 기존 비번 그대로 사용).
- 회원정보 수정 UI.
- 부서 자동완성/검색.

## 3. 사용자 시나리오

### S1. 신규 가입자가 "AI 학습 시작" 선택
1. 가입 폼 완료 → "가입 완료" 클릭.
2. 환영 화면: "환영합니다, {닉네임}님!" + 두 버튼.
3. **"AI 학습 시작하기"** 클릭 → 영상 페이지(`videos`)로 이동.

### S2. 신규 가입자가 "둘러보기" 선택
1. 동일하게 환영 화면 노출.
2. **"홈에서 둘러보기"** 클릭 → 홈(`home`)으로 이동.

### S3. 비번 정책
1. 가입 폼에서 비번 입력 — 안내 텍스트 "8~16자, 영문/숫자/특수문자 각 1개 이상".
2. 정책 미달이면 클라이언트 측 즉시 에러 표시. 서버에서도 동일 검증.

## 4. 기능 요구사항

### F1. 라벨/플레이스홀더 변경
- `components/WelcomePopup.tsx` Step `signup`:
  - 부서 라벨: `부서(팀) *` → `부서(브랜드/팀) *`
  - 직무 플레이스홀더: `예: 사원 / 대리 / 과장` → `예: 상품기획, 생산, OO OO점 점장, 지점 인사팀`

### F2. 비번 정책 강화
- `lib/password.ts`:
  - `isValidSimplePassword` 정규식 변경:
    - 길이: 8~16자
    - **반드시 영문 1개 이상**: `(?=.*[A-Za-z])`
    - **반드시 숫자 1개 이상**: `(?=.*\d)`
    - **반드시 특수문자 1개 이상**: `(?=.*[!@#$%^&*()_+\-=\[\]{};':"\\|,.<>\/?])`
  - 함수명 그대로 유지(`isValidSimplePassword`) — 호출처 변경 없음.
- `WelcomePopup.tsx`:
  - 라벨: "간편 비밀번호 (4~12자, 영문/숫자) *" → "비밀번호 (8~16자, 영문·숫자·특수문자 포함) *"
  - 플레이스홀더: "예: 1234" → "예: Eland@2026"
  - input maxLength: 12 → 16
  - 클라이언트 사전 검증 정규식 동일 패턴 사용.
- 서버 `app/api/users/route.ts`: 에러 메시지 갱신.

### F3. 가입 완료 환영 화면
- `WelcomePopup.tsx`:
  - 신규 step `'welcome'` 추가. 가입 성공 후 `onClose()` 호출하지 않고 `setStep('welcome')` 으로 전환.
  - 화면 구성:
    - 큰 ✨ 또는 🎉 이모지 + "환영합니다, {nickname}님!"
    - 보조 텍스트: "지금 바로 학습을 시작해 보세요."
    - **버튼 1**: "AI 학습 시작하기" (primary, 영상 페이지로)
    - **버튼 2**: "홈에서 둘러보기" (ghost, 홈으로)
- `onClose` prop 시그니처 변경: `onClose: (target?: 'home' | 'videos') => void`.
- `app/page.tsx`:
  - `handleWelcomeClose(target?: 'home' | 'videos')` — `target === 'videos'`이면 `setActiveTab('videos') + 스크롤 상단`.

### F4. 로그인 흐름은 영향 없음
- 로그인 성공 후엔 그대로 `onClose()` (홈으로). 환영 화면 미노출.

## 5. UX

```
환영합니다, 김캠퍼스님!
이제 AI 학습을 시작해보세요.

[ AI 학습 시작하기 ]
[ 홈에서 둘러보기 ]
```

## 6. 데이터 / 스키마

- DB 변경 없음.
- 기존 회원(SHA-256 비번 사용자)은 로그인 흐름이 그대로 작동 (정책 강화는 신규 가입에만 적용).

## 7. 엣지 케이스

| 케이스 | 동작 |
|---|---|
| 비번이 8자 미만 | "8~16자로 입력해주세요" |
| 영문 누락 | "영문/숫자/특수문자를 각각 1개 이상 포함해주세요" |
| 특수문자 누락 (영숫자만) | 동일 메시지 |
| 16자 초과 | maxLength로 입력 차단 |
| 비번 확인 불일치 | 기존 동일 (실시간 표시) |
| 환영 화면에서 새로고침 | 게스트 환영 팝업으로 돌아감(이미 가입은 완료된 상태). 사용자 정보는 localStorage에 저장되어 있어 자동 인식. |
| 모바일 키보드의 특수문자 입력 어려움 | placeholder/help text로 안내, 클립보드 붙여넣기 허용 |
