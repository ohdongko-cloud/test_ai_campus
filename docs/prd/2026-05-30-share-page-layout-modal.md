# PRD: 서비스 공유 페이지 레이아웃 변경 (목록 먼저 + 등록 모달)

- 작성일: 2026-05-30
- 작성자: ohdongko + Claude
- 관련 시스템: `components/SharePage.tsx`, `components/LegalModal.tsx` (참조)
- 영향 버전: v1.0-rc → v1.0-rc+1

## 1. 배경

현재 `/#share` 페이지는 **상단에 등록 폼**, 하단에 **공유된 서비스 목록**이 배치되어 있다.
- 사용자가 페이지에 진입하면 가장 먼저 보이는 게 빈 입력 폼 → "뭘 등록해야 하나?" 머뭇거리는 경험.
- 다른 사람들이 어떤 AI 서비스를 공유했는지 둘러보고 영감을 얻은 뒤 등록하는 자연스러운 흐름을 막음.
- 모바일에서는 폼이 화면 첫 화면을 다 차지해 "이 페이지엔 등록 폼만 있는 듯" 보임.

## 2. 목표 (Goals)

1. **목록 우선 노출**: 페이지 진입 시 공유된 서비스 카드들을 먼저 보여준다 → 영감 + 페이지 가치 즉시 전달.
2. **등록은 명확한 CTA로 분리**: 페이지 하단의 큼직한 "+ 서비스 공유하기" 버튼을 통해 의도적으로 등록 모드 진입.
3. **등록 폼은 모달로 띄움**: 페이지 스크롤을 방해하지 않고 집중된 입력 경험 제공. ESC/백드롭 클릭으로 빠른 취소.
4. **기존 기능은 100% 보존**: 입력 필드, 검증, API 호출, 캐시 우회, 에러 메시지 등 모두 동일.

## 3. Non-Goals

- 카드 디자인/정렬 옵션, 카테고리 필터, 검색 기능 추가는 본 PRD 범위 밖.
- 회원 본인이 등록한 서비스 수정/삭제 기능은 별도 PRD.
- 댓글·좋아요 등 소셜 기능은 본 PRD 범위 밖.
- 우측 하단 플로팅 액션 버튼(FAB)과의 통합/이동은 본 PRD 범위 밖 (FAB는 그대로 둠).

## 4. UX 사양

### 4.1 페이지 본문 (목록 모드)

페이지 구조 (위→아래):
1. 제목 "AI 서비스 공유하기" + 안내 박스(파란 박스 — 등록 안내)
2. (선택) 비로그인 시 안내 박스(앰버 박스)
3. (성공/오류 메시지 영역 — 있을 때만)
4. **공유된 서비스 목록** (헤딩 "공유된 서비스 목록" + 카드 그리드 또는 빈 상태)
5. **하단 CTA 카드**: 큰 "+ 서비스 공유하기" 버튼

### 4.2 하단 CTA 카드 사양

| 항목 | 사양 |
|---|---|
| 위치 | 목록 직후, 페이지 본문 끝 (sticky/fixed 아님 — 자연스러운 흐름) |
| 모양 | 점선 테두리 + 옅은 배경 카드, 가운데 정렬 |
| 본문 | 큰 + 아이콘 + "서비스 공유하기" 텍스트 + 보조 설명 한 줄 |
| 비로그인 시 | 버튼 텍스트 "로그인 후 공유 가능", 클릭 시 안내 메시지 (현 동작 유지) |
| 로그인 시 | 클릭 시 등록 모달 오픈 |

예시:
```
┌──────────────────────────────────────┐
│                                       │
│            ┌──── + ────┐             │
│            │           │             │
│            │  서비스   │             │
│            │  공유하기 │             │
│            └───────────┘             │
│                                       │
│  내가 만든 AI 서비스를 동료와 공유해보세요  │
│                                       │
└──────────────────────────────────────┘
```

### 4.3 등록 모달 (`ShareRegisterModal` 신규)

`LegalModal` 패턴 재활용 (ESC/백드롭 닫기, body 스크롤 잠금, focus 관리).

| 항목 | 사양 |
|---|---|
| 타이틀 | "AI 서비스 공유하기" |
| 보조 텍스트 | "회원 누구나 자유롭게 등록할 수 있습니다. 회사 기밀이나 외부 공개 불가 정보는 등록하지 말아주세요." |
| 입력 필드 | 서비스 이름* / 간단한 설명* / 서비스 URL* / 테스트 계정 정보(선택) — **기존과 동일** |
| 액션 버튼 | 우측 하단 "공유하기" (primary, 검정), 좌측 "취소" (보조, 회색) |
| 모달 너비 | `maxWidth: 560`, `maxHeight: calc(100vh - 32px)` |
| 닫기 동작 | ESC / 백드롭 클릭 / X 버튼 / 취소 버튼 → onClose |
| 등록 성공 시 | 모달 자동 닫기 + 페이지 상단에 success 메시지 표시 + 목록 갱신 |
| 등록 실패 시 | 모달은 **열린 상태 유지**, 모달 상단에 인라인 에러 메시지 (사용자가 입력 수정 가능) |
| 등록 중 (loading) | 공유하기 버튼 disabled + 텍스트 "공유 중..." |

### 4.4 동작 흐름

```
[페이지 진입]
   ↓
공유된 서비스 목록 즉시 노출 (load() 1회)
   ↓
하단 "서비스 공유하기" CTA 카드 표시
   ↓
[CTA 클릭]
   ├─ 로그인 X → 안내 메시지 (현 동작과 동일)
   └─ 로그인 O → 등록 모달 오픈
                  ↓
              필드 입력 + "공유하기" 클릭
                  ├─ 성공(201) → 모달 닫기 + success 메시지 + 목록 갱신
                  └─ 실패(401/429/400/500) → 모달 유지 + 인라인 에러
```

### 4.5 빈 상태

목록이 0건일 때:
- 현재의 "아직 공유된 서비스가 없습니다. 첫 번째로 공유해보세요." 메시지 유지
- 그 아래 CTA 카드는 그대로 노출 → 첫 번째 등록 유도

## 6. 컴포넌트 구조

| 파일 | 변경 유형 | 역할 |
|---|---|---|
| `components/SharePage.tsx` | 수정 | 목록 우선 배치, CTA 카드, 모달 open/close 상태 관리 |

### 6.1 `ShareRegisterModal` Props

```ts
interface Props {
  isOpen: boolean;
  onClose: () => void;
  onSubmitted: (serviceName: string) => void; // 부모가 success 메시지 + load() 실행
}
```

부모(`SharePage`)는 모달 안의 입력 상태에 관여하지 않음. 모달이 자체적으로 입력/검증/POST/에러 처리. 성공 시에만 `onSubmitted(serviceName)` 콜백 호출.

### 6.2 상태 관리 위치

| 상태 | 위치 |
|---|---|
| `services` (목록) | SharePage |
| `successMsg` (페이지 상단 토스트) | SharePage |
| `isLoggedIn` | SharePage |
| `modalOpen` (모달 열림 여부) | SharePage |
| 입력값 4개 (serviceName/description/url/testAccount) | **ShareRegisterModal 내부** |
| `submitting` (등록 중) | ShareRegisterModal 내부 |
| `modalError` (모달 내 에러) | ShareRegisterModal 내부 |

## 7. 접근성 / 반응형

- 모달 열림 시 첫 input(서비스 이름)에 자동 focus
- ESC, 백드롭 클릭, X 버튼, 취소 버튼 → 모두 close
- 모달 열림 시 body scroll 잠금 (`LegalModal` 패턴 재활용)
- 모바일(<640px): 모달 padding 축소, 너비 100%
- 화면 키보드(가상키보드)로 인한 jitter 방지를 위해 `max-height: calc(100vh - 32px)` + `overflow-y: auto`

## 8. 에러 케이스 / 엣지

| 케이스 | 처리 |
|---|---|
| 모달 열린 상태로 페이지 unmount | useEffect cleanup에서 body.style.overflow 복원 |
| 비로그인 사용자가 CTA 클릭 | 모달 열지 않고 페이지 상단에 안내 메시지 (현 동작과 동일) |
| 모달에서 POST 실패 (401/429/400) | 모달 유지, 모달 상단 빨강 에러 박스 + 메시지 표시 |
| 등록 중에 사용자가 모달 닫기 시도 | 닫기 허용 (이미 in-flight 요청은 진행 — 결과는 무시) |
| 같은 이름 중복 등록 | 현재 unique constraint 없음 → 허용 (현 동작 유지) |
| 모달 열린 상태에서 ESC | 닫힘 + 입력값 폐기 (자동 저장 X) |
| 모달 열린 상태에서 브라우저 뒤로가기 (hash 변경) | 다른 탭으로 이동 시 SharePage unmount → cleanup으로 모달 자동 닫힘 + body 스크롤 복원 |
| 키보드 사용자 Tab 이동 | 첫 input 자동 focus 후 자연스러운 Tab 흐름 (전용 focus trap 불필요 — LegalModal 패턴과 일관) |

## 9. 검증 기준 (Acceptance)

- [ ] 페이지 진입 시 등록 폼이 보이지 않음 (모달 닫힌 상태)
- [ ] 페이지 상단~중단에 공유된 서비스 목록 카드가 노출됨
- [ ] 페이지 하단에 점선 카드 + "+ 서비스 공유하기" 버튼 노출
- [ ] CTA 클릭 시 (로그인 상태) 모달 오픈 + 첫 input focus
- [ ] 모달에서 4개 필드 입력 → "공유하기" → 모달 닫힘 + 목록 즉시 갱신 + 상단에 success 메시지
- [ ] ESC, 백드롭, X 버튼, 취소 버튼 4가지로 모두 모달 닫힘
- [ ] POST 401/429/400 시 모달은 유지, 모달 안에 에러 메시지
- [ ] 비로그인 사용자가 CTA 클릭 → 모달 안 뜨고 안내 메시지
- [ ] body 스크롤이 모달 닫힘 후 정상 복원
- [ ] 모바일 뷰포트(<640px)에서 모달이 잘림 없이 정상 노출
- [ ] 빌드 (`npm run build`) 성공, TypeScript 에러 없음
- [ ] 기존 캐시 우회 동작 유지 (POST 직후 새 서비스 즉시 보임)
