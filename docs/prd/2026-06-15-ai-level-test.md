## 1. 배경 / 문제

현 강의 페이지의 레벨테스트는 **"예/아니오/모름" 자가선택 4단계(새싹·초급·중급·고급)** 로, AI를 "안다고 느끼는지"만 묻는다. 실제로 **잘 아는가(지식)·잘 쓰는가(행동)·본인의 큰 숫자를 바꿨는가(EBG)** 를 구분 측정하지 못한다.

전직원(~1,800명)의 AI 역량을 **객관적으로 진단·레벨화하고, 조직(법인/부서/직무)별로 관리**하려면, ① 지식 퀴즈(함정문항 포함), ② 실제 사용 행동(만든 서비스·코드 품질), ③ EBG 업무에 AI를 쓰는 행동패턴을 측정하는 별도 시스템이 필요하다.

> 코드 품질 채점은 **관리자(오너)가 주1회 클로드코드로 오프라인 리뷰 후 점수를 입력**한다 → 서버에서 외부 LLM을 호출하지 않으므로 데이터 반출 이슈 없이 AI캠퍼스(Vercel+Neon+Blob) 내부에 구현 가능하다.

## 2. 목표 / 비목표

### 목표
- G1. 회원가입 직후(또는 강의 진입 시) **3축(지식10% / 행동50% / EBG20%) 자동 측정** + 정성20%(관리자 수기)로 구성, 자동 80%를 100%로 환산해 사용자에게 **점수 + 레벨(1~10)** 노출.
- G2. 문항은 세부영역별 **풀(10~20문항)에서 적응형 랜덤 출제**, 사용자당 총 20개 내외. **쉬운 문항부터, 낮은 허들에서 걸러지면 조기 종료**.
- G3. 보안·EBG 영역은 **함정/행동탐지 문항**으로 실제 수준·행동패턴을 교묘하게 파악(노골적 금지).
- G4. **코딩점수(행동-질)**: 사용자가 zip/html/서비스 링크 업로드 → 관리자가 주1회 채점 → 점수 반영. 미제출/없음은 0점. 채점 전엔 코딩 제외하고 잠정 산출.
- G5. **월 1회 의무 + 수시 재측정**. 결과는 append(이력) → 전월 대비 **성장률** 산출.
- G6. **관리자 대시보드**: 법인/부서(브랜드)/직무별 인원·점수·레벨·전월·성장률·영역별 점수·정성(이머니)·EBG적용을 매트릭스로 확인·관리(레퍼런스 이미지 구조).

### 비목표
- 자동 LLM 코드리뷰(서버에서 외부 모델 호출) — 관리자 오프라인 채점으로 대체.
- 정성평가 20%(이머니/큰숫자)의 자동 산출 — 관리자 수기 입력만.
- 기존 강의 레벨테스트(새싹·초급…) 로직 — 본 시스템으로 대체 예정이나, 마이그레이션/철거는 본 PRD 후속(공존 가능하게 설계).
- 문항 본문(10~20개×각 영역)의 최종 확정 — 시드 샘플 제공 후 오너가 난이도·배점 조정.

## 3. 사용자 시나리오

### S1. 최초 진단 (회원가입 직후)
1. 가입 완료 → 입장 시 레벨테스트 자동 시작 — **강제(스킵 불가)**.
2. 3축 문항을 **한 문제씩** 풀이(쉬운 것 먼저). 영역에서 연속 오답/저득점이면 그 영역 잔여 고난도 문항 스킵 → 총 20개 내외에서 종료.
3. 마지막에 **(선택)** 코딩 산출물 업로드(zip/html/링크) + 서비스수 입력.
4. 결과 화면: **잠정 점수 + 레벨(1~10)** 노출. "코딩점수는 주1회 채점에 반영됩니다" 안내.

### S2. 코딩 채점 반영 (관리자, 주1회)
1. 관리자 콘솔에서 제출된 코딩 산출물 목록 확인 → 클로드코드로 오프라인 리뷰.
2. 점수 입력 → 해당 사용자 **총점 재산출**(코딩 포함) → 레벨 갱신.

### S3. 재측정 (월 1회 의무 / 수시)
1. 매월 측정 주기 도래 → 사용자에게 재측정 안내(의무).
2. 또는 사용자가 수시로 재응시 → 새 결과가 이력에 append, 전월 대비 성장률 표기.

### S4. 관리자 모니터링
1. 관리자 → "AI 레벨" 탭 → 법인/부서/직무 필터.
2. 매트릭스: 이름·목표·점수(Lv)·전월·성장률 / EBG(이머니·EBG적용) / 행동(코딩·서비스수) / 지식(보안·운영도구·자동화확산) 확인·정성 입력·관리.

## 4. 기능 요구사항

### F1. 문항 풀 & 카테고리 모델
- **카테고리 3개 + 세부영역**:
  - C1 지식(10%): `보안` / `운영도구` / `자동화·확산`
  - C2 행동(50%): `코딩점수(질)` / `서비스수(양)`
  - C3 EBG(20%): `EBG 활용`
- 각 **세부영역별 문항 풀 10~20개**. 문항 속성: `category, area, difficulty(1~5), points(배점), type(ox|single|multi|scale|number|upload), body, options[], correct[], active`.
- 배점은 **난이도별 차등이되 편차 과대 금지**(예: 난이도 1=10p ~ 난이도 5=30p 선형, 관리자 조정 가능).

### F2. 적응형 출제 엔진
- 영역별로 **난이도 오름차순** 제시. 풀에서 랜덤 추출(같은 영역 내 동일 난이도는 셔플).
- **조기 종료 규칙**: 한 영역에서 **하위 난이도(1~2) 연속 오답/저득점**이면 그 영역의 상위 난이도 문항을 **스킵**(어차피 못 풀 가능성). 영역별 최소 N문항 보장.
- 사용자당 **총 20문항 내외**(영역별 상한으로 분배). 조기 종료 시 더 적게.
- 코딩점수(질)는 퀴즈가 아니라 **업로드 1스텝**, 서비스수(양)는 **입력 폼**(만든 수/사용중 수/혼자·공유·사용인원/아는 유료서비스 수).

### F3. 함정·행동탐지 문항 설계 원칙
- **보안(C1)**: 정답이 뻔하지 않게 변형. 예) "실적분석용 엑셀(고객 실명·실거래액 포함)을 그대로 NOA/외부 AI에 업로드해 분석을 맡겼다 — 가장 가까운 평가는?" 식 **상황 제시형**(노골적 OX 금지). `.env`·시크릿 하드코딩·고객 PII 업로드를 **상황 속에 숨겨** 판별.
- **EBG(C3)**: "안다"가 아니라 **"실제로 했는가/빈도"** 를 묻는 행동문항. 예) 고객조사(1·2·3차) 인터뷰/질문 설계에 AI 사용 여부·빈도, 시장·경쟁사 BP 조사에 AI 사용, 7P 정렬·이칼 정리·역기획 목표 설정·트랙터(매뉴얼) 제작에 AI 사용, 본인 과업 기록/자동화, 투입(시간·비용) 대비 ROI 점검 여부. **자가과장 탐지**(행동 빈도와 산출물 유무를 교차로 묻기).
- 난이도/정답은 데이터로만 관리(클라이언트 노출 금지).

### F4. 점수·레벨 산출 (결정론)
```
# 영역 원점수(0~1) = 획득배점 / 출제배점 (조기종료로 미출제분은 0 처리)
area_ratio[a] = earned[a] / served_max[a]   # served_max=0이면 0

# 카테고리 점수(0~100)
C1 = 지식 가중(보안*1 + 운영도구*3 + 자동화확산*3 + 서비스이해*3) / 10 * 100   # 지식 10% 내부 비중
C2svc = 서비스수(양) * 100
C3 = EBG 활용 * 100
behavior = 코딩점수질 * 0.6 + 서비스수양 * 0.4   # 행동 50% = 코딩30 + 서비스20

# 자동 종합(0~100): 지식10 + 행동50 + EBG5 = 65를 100으로 환산
auto = (C1/100*0.10 + behavior*0.50 + C3/100*0.05) / 0.65 * 100   # 사용자 노출 점수

# 코딩 보류 상태(채점 전): 코딩 비중 제외 → 지식10 + 서비스20 + EBG5 = 35 정규화(잠정)
auto_provisional = (C1/100*0.10 + C2svc/100*0.20 + C3/100*0.05) / 0.35 * 100
```
- **전체 비중(100%)**: 지식 10%(보안1·운영3·자동화3·서비스3) / 행동 50%(코딩30·서비스수20) / EBG 5% / **정성 35%(관리자 수기)**.
- **코딩 상태 3종**: `pending`(미채점→정규화에서 제외, 잠정 레벨) / `none`(미제출→0점 확정 포함) / `scored`(관리자 점수 반영).
- **정성 35%(이머니/큰숫자)**: 자동 점수에 미포함. 관리자 매트릭스에만 수기 입력·표시.
- **레벨 1~10**: 환산점수 0~100 → 10구간 매핑(**기본 균등 10점 폭**, 관리자 조정 가능). 결과 화면에 점수+레벨 노출.

### F5. 코딩 산출물 제출·채점 워크플로우
- 사용자: zip/html 파일(Vercel Blob) 또는 서비스 링크 업로드. 없으면 건너뜀(→ `none`).
- 관리자: 제출 큐 → 다운로드/링크 확인 → 오프라인 클로드코드 리뷰 → **점수 입력 + 메모** → 사용자 총점 재산출.
- "주1회 채점" 안내 문구 사용자 결과 화면 고정.

### F6. 재측정 주기·이력
- **월 1회 의무**(주기 도래 시 재측정 유도) + **수시 재응시** 허용.
- 매 응시 결과를 **append**(`level_test_results` 이력). 전월 대비 **성장률(%)** 계산.
- 관리자: 측정 주기(period) 생성·마감 관리.

### F7. 관리자 대시보드 (레퍼런스 이미지)
- 필터: **법인 / 부서(브랜드) / 직무**(기존 `org_units` 재사용).
- 매트릭스 컬럼(이미지 기준):
  - 구분: 부서 · 이름
  - 측정지표: 목표 · **점수(Lv)** · 전월 · 성장률
  - (EBG 40%): 이머니(정성·수기) · EBG적용(=C3)
  - (행동 50%): 코딩점수(질) · 서비스수(양)
  - (지식 10%): 보안 · 운영도구 · 자동화/확산
- 인원별 행, 정성(이머니/목표) 수기 입력, 영역별 점수 표시, 정렬·필터·내보내기.

### F8. 진입·노출
- 회원가입 직후 입장 시 **강제 자동 시작(스킵 불가)** — 완료해야 서비스 이용. 홈 "AI 레벨 진단" 배너(A안, 기구현)와 연결.
- 완료자는 결과 화면 재열람 가능, 미완료자는 재유도(매 진입 시 모달).

### F9. 데이터 모델 스케치 (1차 MVP — 멱등 마이그레이션)
```
-- M007: 문항 풀
lt_questions(
  id UUID PK, category TEXT,            -- 'knowledge'|'behavior'|'ebg'
  area TEXT,                            -- 'security'|'ops'|'automation'|'service_count'|'ebg'
  difficulty INT,                       -- 1~5
  points INT,                           -- 배점
  qtype TEXT,                           -- 'ox'|'single'|'multi'|'scale'|'number'
  body TEXT, options JSONB,             -- 선택지
  correct JSONB,                        -- 정답(서버 전용, 클라이언트 미노출)
  is_trap BOOLEAN DEFAULT false,        -- 함정문항 표시
  active BOOLEAN DEFAULT true,
  created_at, updated_at TIMESTAMPTZ )

-- M008: 응시 세션 + 결과(이력 append)
lt_attempts(
  id UUID PK, user_id UUID, period_id TEXT NULL,
  status TEXT,                          -- 'in_progress'|'done'
  c1_score NUMERIC, c2_score NUMERIC, c3_score NUMERIC,  -- 0~100
  coding_status TEXT DEFAULT 'pending', -- 'pending'|'none'|'scored'
  coding_score NUMERIC NULL,
  auto_score NUMERIC, level INT,        -- 환산점수 0~100, 레벨 1~10
  started_at, finished_at TIMESTAMPTZ )

-- M009: 문항별 응답(채점 근거)
lt_responses(
  id UUID PK, attempt_id UUID FK, question_id UUID,
  answer JSONB, is_correct BOOLEAN, earned INT, served_max INT,
  created_at TIMESTAMPTZ )

-- M010: 서비스수(양) 입력
lt_service_inventory(
  id UUID PK, attempt_id UUID FK,
  made_count INT, inuse_count INT, share_mode TEXT, share_users INT,
  paid_known INT, links JSONB )
```
> 2차(코딩 업로드·채점 `lt_coding_submissions`), 3차(정성 `lt_qualitative`)는 해당 단계에서 추가. 모두 `ADD ... IF NOT EXISTS`·기본값으로 하위호환.

## 5. UX / 디자인
- 한 문제씩(기존 LevelTest UX 계승), 상단 진행바(최대 20문항, 적응형 단축 안내).
- 결과 화면: 큰 점수 + 레벨(1~10) + 3축 막대 + "가장 약한 영역 → 추천 강의" + "코딩점수 주1회 반영" 안내 + (선택)코딩 업로드.
- 관리자 매트릭스: 신호등/밀집표 스타일(소규모 인원 가독), 법인·부서·직무 드롭다운.
- 문항 문구는 70대도 이해할 쉬운 한국어(EBG 가이드 어투 원칙).

## 6. 엣지 케이스

| 케이스 | 동작 |
|---|---|
| 코딩 미제출 | 코딩=0(`none`) 확정 포함, 그 외 영역으로 잠정→확정 |
| 코딩 채점 전 | `pending` → 코딩 비중 제외 정규화(잠정 레벨), 안내 노출 |
| 조기 종료(저활용자) | 미출제 고난도=0, 20문항 미만 종료 — 정상 |
| 재응시 다회 | 모두 append, 최신값을 현재 레벨로, 전월=직전 주기값 |
| 인원 적은 조직 성장률 | 분모 0/소표본 → 성장률 'N/A' 처리(0으로 오해 금지) |
| 문항 풀 부족(영역<요구수) | 있는 만큼만 출제, served_max 기준 정규화 |
| 함정문항 '모름' 선택 | 0점(오답과 구분 기록 가능) |
| 직군/부서 미지정 사용자 | '미분류'로 집계, 관리자에서 보정 |
| 동일 사용자 동시 응시(중복탭) | 진행 중 attempt 1개로 잠금 |
| PII 업로드(코딩 zip에 고객데이터) | 업로드 고지 + 관리자만 접근, 응답/로그/Sentry에 내용 미노출 |

## 7. 성공 기준
- [ ] 가입 후 입장 시 레벨테스트 자동 시작, 한 문제씩 진행.
- [ ] 적응형: 저난도 연속 오답 시 해당 영역 조기 종료, 총 20문항 내외.
- [ ] 보안/EBG 함정·행동문항이 노골적이지 않게 동작(정답 데이터 클라이언트 미노출).
- [ ] 코딩 제외 잠정 점수 → 관리자 채점 입력 시 총점·레벨 재산출.
- [ ] 자동 80% → 100% 환산, 점수+레벨(1~10) 결과 노출.
- [ ] 월 1회/수시 재측정, append 이력 + 전월 성장률.
- [ ] 관리자에서 법인/부서/직무별 매트릭스(점수·전월·성장률·영역별·정성) 확인·입력.
- [ ] PII 보호(§6), 권한 체크, 레이트리밋, 에러 통일.
- [ ] `npx tsc --noEmit` + `npm run build` 통과.
