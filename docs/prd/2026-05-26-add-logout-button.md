# PRD: 메인 페이지 로그아웃 버튼

- 작성일: 2026-05-26
- 작성자: Claude (요청자: <오너>)
- 범위: 클라이언트 UI 추가 + 기존 `/api/users/logout` 호출

---

## 1. 배경 / 문제

회원가입/로그인 흐름은 완성됐지만 **로그아웃 UI가 없다**. 사용자가 한 번 가입/로그인하면 서버 세션 쿠키(user_session)와 localStorage(UserInfo) 둘 다 그대로 남아 다른 계정 전환이나 명시적 종료가 불가능.

## 2. 목표 / 비목표

### 목표
- G1. 메인 페이지 헤더에 **로그인 상태일 때만** "로그아웃" 버튼 노출.
- G2. 클릭 시: 서버 세션 쿠키 삭제(`POST /api/users/logout`) + 클라이언트 localStorage `UserInfo` 클리어 + 환영 팝업 재노출.
- G3. PC 헤더와 모바일 드로어 둘 다 지원.

### 비목표
- 로그아웃 확인 모달(불필요한 마찰).
- "모든 기기에서 로그아웃" (세션 토큰 블랙리스트 — 추후).
- 어드민 로그아웃(이미 `onExit`에서 처리됨).

## 3. 사용자 시나리오

### S1. PC 로그아웃
1. 사용자 chip 옆 "로그아웃" 텍스트 버튼 클릭.
2. 서버 세션 쿠키 삭제 → localStorage UserInfo 초기화.
3. 페이지가 환영 팝업(WelcomePopup) 상태로 전환.

### S2. 모바일 로그아웃
1. 햄버거 메뉴 → 하단 "로그아웃" 항목 클릭.
2. S1과 동일.

## 4. 기능 요구사항

### F1. PC 헤더
- 위치: 사용자 chip 우측, "관리자" 버튼 좌측.
- 노출 조건: `userInfo` 가 존재 (== 로그인됨).
- 라벨: "로그아웃" (텍스트만, 작은 secondary 버튼).
- 스타일: 기존 "관리자" 버튼과 유사한 ghost 스타일.

### F2. 모바일 드로어
- 위치: 기존 "관리자 모드" 버튼 위 (또는 그 아래).
- 노출 조건: `userInfo` 존재.
- 동일 핸들러 사용.

### F3. 로그아웃 핸들러
- `POST /api/users/logout` 호출 (에러는 무시 — 어차피 클라이언트 정리 진행).
- `localStorage.removeItem('axtf_user_info')` (또는 기존 키).
- `setUserInfo(null)` → 환영 팝업이 다시 표시되도록 `showWelcome=true`.
- 현재 탭은 'home'으로 리셋.

## 5. UX / 디자인

```
[사용자 chip] [로그아웃] [관리자] [≡]
```
- 로그아웃 버튼은 사용자 chip 바로 옆에 배치해 "내 계정 영역"으로 묶이는 시각적 그룹화.
- 모바일은 햄버거 드로어 안의 텍스트 메뉴.

## 6. 엣지 케이스

| 케이스 | 동작 |
|---|---|
| 서버 logout API 실패 (네트워크) | 클라이언트 상태는 정리 (cookie는 만료될 때까지 남지만 클라이언트 측 표시는 게스트) |
| 로그인 안 된 상태 | 로그아웃 버튼 자체가 안 보임 |
| 어드민 모드 중 메인 페이지 로그아웃 시도 | 메인 페이지가 아닌 어드민 페이지이므로 해당 없음 |
| 자동로그인 쿠키 (30일) 보유자 | 서버에서 쿠키 삭제하므로 다음 진입 시 다시 로그인 필요 |
