## 1. 배경

직전 변경(`4aa2852`)으로 영상 모달이 크기 토글(컴팩트/표준/와이드)을 제공하고 정보 영역은 항상 노출되도록 했다. 그러나 정보 영역 안에 **영상 정보 + 스테이지(학습 단계) + 댓글**이 모두 세로로 쌓여 있어:

1. **영상을 보면서 스테이지를 함께 확인하기 어렵다** — 영상 아래로 스크롤해야 스테이지가 보임.
2. **스테이지가 가장 중요한 학습 자료인데 영상 정보·댓글에 묻힘** — 사용자가 "다음 단계로 뭘 따라할지" 파악이 느림.
3. **와이드 모드에서 영상 옆 좌우 검정 띠 공간이 낭비**됨 — 16:9 영상 옆 빈 공간을 활용할 여지.

## 2. 목표 (Goals)

1. **스테이지를 모달 우측 사이드바로 분리**해 영상 시청 중에도 항상 보이게 한다.
2. **사이드바 자체 스크롤** — 스테이지가 많아도 영상 영역을 침범하지 않음.
3. **접었다 폈다 토글** — 사용자가 영상에만 집중하고 싶을 때 사이드바를 숨길 수 있게.
4. **모달 크기 토글(컴팩트/표준/와이드)과 자연스럽게 공존** — 기존 토글 보존.
5. **모바일에서도 작동** — 좁은 화면에서는 오버레이 방식으로 슬라이드.

## 3. Non-Goals

- 스테이지 정렬/필터/검색 기능은 본 PRD 범위 밖.
- 스테이지를 별도 모달로 띄우는 옵션은 본 PRD 범위 밖.
- 댓글을 사이드바로 분리하는 것은 본 PRD 범위 밖 (댓글은 기존대로 본문 하단).
- 사이드바 좌/우 위치 선택 옵션은 본 PRD 범위 밖 (우측 고정).

## 4. UX 사양

### 4.1 데스크탑 (≥ 768px)

모달 구조 (가로 2분할):

```
┌──────────────────────────────────────────────────────────┐
│ 영상 영역 (16:9, maxHeight 사이즈 토글에 따라)             │
├──────────────────────────────────────────────────────────┤
│ [컴팩트] [표준] [와이드]   ← 사이즈 토글 바                │
├──────────────────────────────────────────────────────────┤
│ 🔒 본 영상은 이랜드리테일 사내 한정 자료입니다.            │
├────────────────────────────────────┬─────────────────────┤
│                                    │ ◀  학습 단계 (3)    │
│ 영상 정보 (제목/설명/뱃지)         │ ─────────────────── │
│                                    │ ┌─────────────────┐ │
│ 댓글                               │ │ Step 1          │ │
│ ...                                │ │ 설명...         │ │
│                                    │ └─────────────────┘ │
│ (스크롤)                           │ ┌─────────────────┐ │
│                                    │ │ Step 2          │ │
│                                    │ │ ...             │ │
│                                    │ └─────────────────┘ │
│                                    │ (스크롤)            │
└────────────────────────────────────┴─────────────────────┘
```

| 항목 | 사양 |
|---|---|
| 사이드바 위치 | 모달 본문 영역의 **우측** (영상 + 사이즈 토글 + 한정 안내 바로 아래의 가로 2분할 영역) |
| 사이드바 너비 (펼침) | 320px (고정) |
| 사이드바 너비 (접힘) | 36px (토글 핸들만 보이는 좁은 띠) |
| 토글 핸들 | 사이드바 좌측 가장자리에 세로로 붙은 클릭 영역. 펼침: `▶ 닫기`, 접힘: `◀ 학습 단계 (N)` |
| 사이드바 헤더 | "학습 단계 (N개)" + 닫기 버튼 (X 또는 ▶) |
| 사이드바 본문 | 스테이지 카드 리스트 (현재 카드 디자인 그대로) + 자체 `overflow-y: auto` 스크롤 |
| 좌측 본문 영역 | 영상 정보 + 댓글 (스테이지 섹션 제거됨) + 자체 스크롤 |
| 사이드바 ↔ 좌측 본문 구분선 | 1px solid `T.border` |
| 스테이지 0개인 영상 | 사이드바 자동 숨김 — 토글 핸들도 안 보이고 본문이 전체 너비 사용 |

### 4.2 모바일 / 좁은 화면 (< 768px)

| 항목 | 사양 |
|---|---|
| 사이드바 기본 상태 | **접힘** (좁은 화면에서 영상 시청을 방해하지 않게) |
| 토글 핸들 | 좌측 본문 영역 우상단에 떠 있는 작은 칩 — "학습 단계 (N)" |
| 펼침 시 동작 | 사이드바가 본문 위로 **슬라이드 오버레이** (모달 우측에서 안쪽으로) |
| 펼침 폭 | 모달 너비의 85% (배경은 반투명 백드롭 — 모달 내부에서만) |
| 닫기 | 토글 핸들 다시 클릭 / 백드롭 클릭 / ESC |

### 4.3 사이드바 상태 관리

| state | 의미 | 초기값 |
|---|---|---|
| `sidebarOpen: boolean` | 사이드바 펼침 여부 | **데스크탑**: 스테이지 ≥ 1개면 `true`, 0개면 `false` / **모바일**: 항상 `false` |
| 사용자가 토글한 상태 | 모달 닫고 다시 열 때 유지 안 해도 됨 (모달 재오픈 시 초기값으로) | - |

### 4.4 사이즈 토글과의 상호작용

- 사이즈 토글(컴팩트/표준/와이드)은 **모달 전체 너비**를 변경. 사이드바가 펼쳐져 있어도 그대로 작동.
- 사이드바 펼침 시 좌측 본문 영역의 가용 너비 = 모달 너비 - 320px.
- 와이드(1280px) + 사이드바 펼침 → 좌측 본문 ≈ 960px. 영상은 maxHeight 70vh로 별도 제약(영상은 좌측 본문 위쪽에 자리, 사이드바와 같은 행이 아님 → §4.5 참조).

### 4.5 영상 영역 vs 사이드바 — 같은 행인가?

**아니오.** 영상은 **모달 최상단**에 가로 100% 차지 (현재와 동일). 사이드바는 **영상 아래 본문 영역에만** 적용 (영상은 사이드바 영향 안 받음).

근거:
- 영상이 사이드바와 같은 행이면 영상 폭이 줄어 16:9 비율 유지가 어려움(시청 경험 ↓).
- 사이드바는 "학습 보조"이므로 영상 시청 위치(상단)는 고정, 사이드바는 정보 영역만 분할.
- 사용자가 와이드 모드 + 사이드바 펼침 시: 영상은 1280px 그대로, 본문 영역만 좌(960px) + 우(320px) 분할.

### 4.6 빈 상태

- **스테이지 0개**: 사이드바 자체가 안 뜸. 좌측 본문이 전체 너비. 토글 핸들도 안 보임.
- **스테이지 1개 이상**: 데스크탑 펼침 / 모바일 접힘으로 시작.

| 파일 | 변경 |

`sidebarOpen` state 신규. 미디어 쿼리 또는 viewport width 측정으로 데스크탑/모바일 동작 분기.

### 6.1 신규 state
```ts
const [sidebarOpen, setSidebarOpen] = useState<boolean>(() => {
  // 초기값: 데스크탑 + 스테이지 있음 → true, 그 외 false
  if (typeof window === 'undefined') return false;
  return window.innerWidth >= 768 && (selectedVideo?.stages?.length ?? 0) > 0;
});
```

비고: `selectedVideo`가 바뀔 때(=다른 영상 선택) `sidebarOpen`을 재계산해야 함 → `useEffect([selectedVideo?.id])`로 갱신.

### 6.2 토글 동작
- 핸들 클릭 → `setSidebarOpen(o => !o)`
- 모바일 오버레이 백드롭 클릭 → `setSidebarOpen(false)`
- ESC 키 → 모달 닫기(현행 유지) — 사이드바만 닫지 않음 (사용자가 모달 자체를 닫고 싶을 가능성이 더 큼)

## 8. 에러 케이스 / 엣지

| 케이스 | 처리 |
|---|---|
| 스테이지가 0개인 영상 | 사이드바 자체 미렌더 (핸들 포함). 좌측 본문 전체 너비. |
| 사이드바 펼침 + 좁은 화면(돌발) | resize 이벤트로 viewport width 추적, 768px 이하로 줄어들면 자동 접힘 + 오버레이 모드로 전환 |
| 모달 크기 토글 변경 (컴팩트 ↔ 와이드) | 사이드바 상태는 유지, 좌측 본문 너비만 자동 변경 |
| 사이드바 펼침 상태에서 다른 영상 선택 | `useEffect([selectedVideo?.id])`로 신규 영상 스테이지 수에 맞게 초기값 재계산 |
| 사이드바 펼침 + 영상 정보 매우 길어 좌측 본문 스크롤 발생 | 좌측 본문 자체 `overflow-y: auto` — 사이드바 영향 없음 |
