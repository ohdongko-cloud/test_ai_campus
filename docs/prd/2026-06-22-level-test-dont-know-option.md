# PRD: 레벨테스트 "잘 모름" 선택지 + 코딩 "없어요" 확정

- 작성일: 2026-06-22
- 작성자: Claude (요청자: <오너>)
- 범위: 지식 문항에 "잘 모름" 옵션 추가(셔플 제외·항상 마지막) + 코딩 제출 단계에 "만든 서비스 없어요" 버튼(코딩 0점 확정→재산출). DB 스키마 변경 없음(기존 컬럼 사용).

---

## 1. 배경 / 문제
지식 문항(보안·운영도구·자동화·서비스이해)은 **4지선다 정답형**이라 모르는 사용자도 **찍어야** 한다 → 추측 정답이 실력을 과대평가한다. 또 코딩 제출 단계는 "건너뛰기"(=나중에)만 있어, **만든 서비스가 없는** 사용자가 "없음(0점 확정)"을 명확히 선택할 길이 없다.

## 2. 목표 / 비목표
### 목표
- G1. 모든 **지식 문항**에 **"잘 모름"**(0점) 선택지 추가 → 정직한 미응답 측정.
- G2. "잘 모름"은 **보기 셔플에서 제외, 항상 마지막**에 고정(찾기 쉽게).
- G3. 코딩 제출 단계에 **"만든 서비스 없어요"** 버튼을 하단 제출 버튼 근처에 잘 보이게 추가. 누르면 **코딩=없음(0점) 확정** → 총점 재산출.
### 비목표
- 행동·EBG 문항 변경(이미 "0개/0건" 최저 옵션이 "없음" 역할 — 유지).
- 채점 가중치·문항 내용 변경.
- DB 스키마 변경(coding_status='none'은 기존 컬럼).

## 3. 사용자 시나리오
### S1. 모르는 지식 문항
1. 지식 문항 4지선다 + 맨 아래 **"잘 모름"**.
2. "잘 모름" 선택 → 0점(오답과 동일) → 적응형상 그 영역 상위 난이도 조기 종료.

### S2. 만든 서비스 없음
1. 코딩 제출 단계 하단: [건너뛰기] [만든 서비스 없어요] [제출하고 결과 보기].
2. "만든 서비스 없어요" → 코딩 **없음(0점) 확정** → 결과(코딩 포함 총점·레벨).
3. (대비) "건너뛰기" → 코딩 보류(pending), 잠정 점수(기존).

## 4. 기능 요구사항
### F1. 지식 문항 "잘 모름" (lib/level-test-questions.ts)
- 지식 헬퍼 `K()`가 옵션 배열 끝에 `"잘 모름"`을 자동 추가. `correct` 인덱스는 기존 4개 중이라 불변.
- 행동/EBG 헬퍼 `B()`는 변경 없음.

### F2. 셔플 제외·마지막 고정 (lib/level-test-engine.ts)
- `optionPerm`: 지식 문항(정답형)에서 **마지막 옵션이 "잘 모름"이면 그 인덱스를 고정**하고 나머지(0..n-2)만 셔플 → "잘 모름" 항상 마지막 표시.
- 채점: "잘 모름" 표시 선택 → 원본 마지막 인덱스 → `correct`에 없음 → 0점.

### F3. 코딩 "만든 서비스 없어요" 버튼 (components/AiLevelTest.tsx)
- 코딩 단계 하단 버튼 영역: 기존 [건너뛰기][제출] 사이/옆에 **[만든 서비스 없어요]** 추가(시인성 위해 제출 버튼 근처).
- onClick → `POST /api/ai-level-test/coding` (FormData `none=1`) → 성공 시 `setPhase('result')`.

### F4. 코딩 없음 확정·재산출 (app/api/ai-level-test/coding/route.ts)
- body(form) `none==='1'`이면: 파일/링크/Blob 처리 생략. 사용자 최신 `ai_level_attempts` 1행을 `coding_status='none', coding_score=0`로 두고 **`recomputeWithCoding(c1,c2,c3, 0)`** 로 `auto_score`/`level` 재산출·UPDATE. 제출물 row(ai_level_coding) 생성 안 함.
- 세션 필수(기존), 레이트리밋(기존), 에러 통일.

## 5. UX / 디자인
- "잘 모름": 일반 옵션과 동일 스타일, 맨 아래. (정답 강조 없음)
- 코딩 버튼: "만든 서비스 없어요"는 secondary(건너뛰기와 구분되되 제출 옆에서 눈에 띄게). 누르면 즉시 결과로.
