### 14. 콘텐츠 보호 강화 (강의 영상 유출 억제) — *향후 검토*
> 배경: F12 차단·확장프로그램(Claude/GPT 애드온) 차단 가능 여부 문의 (2026-06-04).
> **결론: "완전 차단"은 기술적으로 불가능.** 현실 목표는 "캐주얼 유출 억제 + 추적".

- 핵심 사실 (의사결정 기록)
  - **브라우저 확장 차단 불가** — 확장은 웹페이지보다 상위 권한. 페이지가 막거나 감지할 공식 API 없음. AI 확장의 화면 읽기는 사용자가 직접 캡처/복사하는 것과 동일하여 차단 불가.
  - **F12/개발자도구 완전 차단 불가** — 단축키 차단·우클릭 차단·devtools 감지는 모두 "억제"일 뿐, 브라우저 메뉴·JS 비활성화·타 브라우저로 우회 가능.
  - **쿠팡/네이버 사례**: 실제론 ① 안티디버깅(`debugger` 루프)+devtools 감지로 "반응"시키고 ② **서버측 상용 안티봇(Akamai 등)+동적 암호화 데이터**로 스크래퍼에게 알맹이를 안 줌. 클라이언트 차단이 아니라 **서버 방어 + 억제**. 기업급 유료·전담 운영이 필요 → 사내 소규모 포털엔 과함.
- 현재 적용된 보호 (유지)
  - ✅ 시청자 이메일 **워터마크**(유출 시 추적) — *가장 실효적*
  - ✅ "외부 공유·녹화·캡처 금지" 고지 + 우클릭/복사 일부 차단
  - ✅ 안드로이드 앱 FLAG_SECURE(스크린샷 차단)
- 추후 검토 가능 (효과 대비 낮음, 우선순위 낮음)
  - [ ] F12·`Ctrl+Shift+I/J`·`Ctrl+U` 단축키 차단 + 우클릭 차단 (억제용)
  - [ ] 개발자도구 열림 감지 → 영상 블러 + 경고 오버레이
  - [ ] (고급) 영상 원본 URL 비노출 / 서명된 단기 URL / DRM — 비용·복잡도 큼
  - 주의: 위 억제책은 정상 사용자·접근성에 불편을 줄 수 있어 **도입 시 트레이드오프 명시 필요**.
