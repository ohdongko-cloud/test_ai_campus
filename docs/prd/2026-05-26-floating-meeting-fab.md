## 1. 배경

현재 우측 하단 플로팅 액션은 **안드로이드 앱**(조건부) + **소통방 입장**(상시) 2개. 멘토링 예약은 홈 화면 카드 1곳에서만 진입 가능 → 다른 탭에서 곧장 진입 어려움.

## 2. 목표 / 비목표

### 목표
- G1. 미팅요청 FAB 추가 (안드로이드 앱 버튼 위, 스택 최상단).
- G2. 라벨 **"미팅요청"** 작게 표시 (기존 두 버튼과 동일 포맷).
- G3. 클릭 → 멘토링 예약 페이지(`#meeting` 탭) 이동 (외부 링크 X, SPA 내부 라우팅).
- G4. 트렌디한 SVG 아이콘 (생성).

### 비목표
- 멘토링 슬롯 quick-view (예약 페이지 진입만).
- 알림 배지(읽지 않은 답신 수 등).

## 4. 디자인 — 신규 SVG 아이콘

**컨셉**: 캘린더 + 체크마크. "슬롯 예약/확정"을 직관적으로 전달.

**스타일**:
- 흰색 라인 아이콘 (오렌지 배경 위)
- 둥근 모서리 라운드, 굵기 2px, 친근한 톤
- 32×32 viewBox

**구성**:
- 캘린더 본체 (사각형 + 상단 가로선 분리)
- 캘린더 상단 작은 링 2개 (걸이 표현)
- 본체 내부 체크마크 ✓

**배경 컬러**: `#FF914D` (Eland 보조 컬러, 따뜻함·인간적인 톤)

## 5. 기능 요구사항

### F1. FloatingActions 확장
- prop `onNavigate?: (tab: 'meeting') => void` 추가.
- onNavigate가 주어지면 미팅요청 버튼 노출. 누락 시 미표시.
- 클릭 핸들러: `addClickLog('미팅요청 FAB')` + `onNavigate('meeting')`.

### F2. app/page.tsx
- `<FloatingActions onNavigate={navigateTo} />` 로 prop 전달.
- 어드민 모드 진입 시 비표시 (기존 동일).

### F3. 라벨/접근성
- 라벨: `미팅요청`
- aria-label / title: `멘토링 예약 페이지로 이동`

## 6. 엣지 케이스

| 케이스 | 동작 |
|---|---|
| 이미 미팅 탭에 있을 때 클릭 | navigateTo 호출되어 상단 스크롤만 발생. 사용자 부담 없음. |
| 모달이 열린 상태에서 클릭 | z-index 999 < 모달(1000)이라 가려짐. 의도된 동작. |
| 모바일 작은 화면 | 3개 스택해도 56px × 3 + gap = 약 200px → 화면 우측 하단에 적정. |
| 안드로이드 앱 URL 미설정 시 | 미팅요청 + 소통방 2개만 스택 (자연스러운 노출). |
