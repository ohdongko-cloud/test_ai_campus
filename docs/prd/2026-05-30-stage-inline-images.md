# PRD: 스테이지 인라인 이미지 (펼침 시 자동 노출)

- 작성일: 2026-05-30
- 작성자: <오너> + Claude
- 관련 시스템: `videos.stages` JSONB, `components/AdminVideos.tsx`(StageEditor), `components/VideoPage.tsx`(stagesPanel)
- 영향 버전: v1.0-rc → v1.0-rc+1

## 1. 배경

현재 강의 영상 스테이지는 **제목 + 설명(텍스트)** 만 표시한다. 사용자가 따라하면서 화면 캡처/도식/스크린샷이 필요한 경우가 많은데:
- 첨부파일 다운로드 → 다른 창에서 열기 → 학습 흐름 끊김
- 텍스트만으로 설명하기 어려운 단계(예: "이 버튼 클릭" → 이미지 한 장으로 즉시 이해)

**스테이지 펼침 즉시 이미지가 인라인 노출**되어야 학습 효율이 높음.

## 2. 목표

1. 어드민이 스테이지 편집 시 **이미지 여러 장 첨부** 가능
2. 사용자가 스테이지 펼침 시 **이미지가 본문에 인라인으로 즉시 표시**
3. 이미지 클릭 시 라이트박스(확대 보기)로 원본 크기 확인

## 3. Non-Goals

- 동영상 임베드 (현재 영상은 YouTube로 충분)
- 이미지 자체 편집/크롭 (외부 도구 사용 후 업로드)
- 이미지 캡션·alt 텍스트 직접 입력 (파일명 자동 사용)

## 4. UX 사양

### 4.1 어드민 — 스테이지 편집 (`StageEditor`)

각 스테이지 편집 카드 안에 **"📷 이미지" 영역** 추가 — 제목/설명 입력 옆 또는 아래:

```
┌─ Step 1 ──────────────── [▲▼] [✕ 삭제] ─┐
│ 제목: [PRD 생성 프롬프트                ] │
│                                          │
│ 설명: [전체 복사 후 AI에...              ] │
│       [                                  ] │
│                                          │
│ 📷 이미지 (2)                            │
│ ┌─────┐ ┌─────┐  ┌─────────────────┐     │
│ │[img]│ │[img]│  │ ➕ 이미지 추가  │     │
│ │  ✕  │ │  ✕  │  └─────────────────┘     │
│ └─────┘ └─────┘                          │
└──────────────────────────────────────────┘
```

| 항목 | 사양 |
|---|---|
| 위치 | 스테이지 카드 본문, 설명 입력 아래 |
| 썸네일 크기 | 80×80, 둥근 모서리 8px |
| 정렬 | 가로 wrap (모바일은 자동 4열 → 2열) |
| 추가 버튼 | 점선 카드 (파일 선택 / 드래그&드롭) |
| 삭제 | 썸네일 우상단 ✕ 버튼 (1클릭 즉시 삭제) |
| 순서 | 업로드 순서 그대로 (드래그 정렬은 v2) |
| 한도 | 스테이지당 최대 10장, 한 장당 10MB |
| 허용 형식 | PNG, JPEG, WEBP, GIF |

### 4.2 사용자 — 영상 모달 사이드바 학습 단계 탭

스테이지 펼침 시 기존 설명 텍스트 **아래에 이미지 그리드** 자동 노출:

```
[Step 1] PRD 생성 프롬프트              ▼ (펼침)
─────────────────────────────────────────
[전체 복사] 버튼

전체 복사 후 AI에 붙여 넣어주세요...
(스테이지 설명 본문 텍스트)

┌─ 이미지 ──────────────────────────────┐
│ ┌──────┐ ┌──────┐                    │
│ │[img1]│ │[img2]│  ← 클릭 시 라이트박스 │
│ └──────┘ └──────┘                    │
└──────────────────────────────────────┘
```

| 항목 | 사양 |
|---|---|
| 위치 | 스테이지 설명 텍스트 직후 |
| 표시 모드 | 인라인 그리드 (썸네일 120×120) |
| 클릭 동작 | 라이트박스 모달 — 원본 크기 + 좌/우 화살표로 이동 |
| 라이트박스 닫기 | ESC, 백드롭 클릭, X 버튼 |
| 이미지 없음 | 그리드 자체 미노출 (기존 텍스트만) |
| 외부 유출 차단 | 영상 보호 정책 동일 적용: 우클릭/드래그 차단, 워터마크 적용은 v2 |

### 4.3 데이터 구조

`videos.stages` JSONB 안의 각 stage 객체에 **`images: string[]`** 필드 추가:

```ts
// 변경 전
{ id: "...", title: "...", description: "..." }

// 변경 후
{ id: "...", title: "...", description: "...", images?: string[] }
```

- `images`: Vercel Blob URL 배열
- optional — 기존 스테이지는 `images` 누락 시 빈 배열로 동작 (역호환)
- DB 스키마 변경 없음 (JSONB 안 필드만 추가)

## 5. 인프라

- **Vercel Blob** 재사용 (첨부파일과 동일 스토리지)
- Blob pathname 규칙: `videos/${videoId}/stages/${stageId}/${timestamp}_${원본명}`
- env `BLOB_READ_WRITE_TOKEN` (첨부 작업에서 이미 설정 안내됨)

### 비용 추정
- 영상 50개 × 스테이지 평균 5개 × 이미지 평균 3장 × 평균 500KB
- = 약 **375MB** → 무료 한도(5GB) 내 충분

## 6. API 설계

### 6.1 `POST /api/admin/videos/[id]/stages/[stageId]/images`
- **권한**: `requireAdmin('videos')`
- **Body**: multipart/form-data (file 필드)
- **응답**: `{ url: string }` — Blob 공개 URL
- **흐름**:
  1. 영상 존재 + 권한 확인
  2. MIME 화이트리스트 (image/png, image/jpeg, image/webp, image/gif)
  3. 10MB 한도 체크
  4. 스테이지 안 이미지 개수 ≤ 10 확인
  5. Blob `put()` → URL 반환
  6. **DB 업데이트는 클라이언트가 stages 전체 저장 시 일괄 처리** (기존 패턴 유지)

> 💡 **중요**: 스테이지 자체는 이미 `PATCH /api/admin/videos/[id]` 에서 stages JSONB 통째로 저장하는 구조. 이미지 URL은 클라이언트가 stage.images 배열에 추가한 뒤 stage 저장 시 함께 영구화.

### 6.2 `DELETE /api/admin/videos/[id]/stages/images`
- **권한**: `requireAdmin('videos')`
- **Body**: `{ url: string }` — 삭제할 Blob URL
- **흐름**: Blob `del()` → 200. DB 업데이트는 클라이언트가 별도.

### 6.3 사용자 다운로드 API
**없음** — 이미지는 Blob 공개 URL을 클라이언트가 직접 `<img src>` 로 사용. 사내 한정이지만 영상이 YouTube unlisted 인 것과 동일 수준(URL 추측 불가능한 random suffix 의존).

## 7. 보안

- **업로드 권한**: 어드민(`videos` 권한)만
- **MIME 화이트리스트**: image/png|jpeg|webp|gif (svg 제외 — XSS 위험)
- **크기 한도**: 10MB / 장, 10장 / 스테이지
- **외부 유출**: Blob URL이 public이지만 random suffix로 추측 불가. 영상 보호 정책(우클릭/드래그 차단) 사용자 페이지에서 동일 적용.
- **악성 파일**: MIME 화이트리스트로 1차 차단, Blob도 Content-Type 그대로 서빙 (실행 불가)

## 8. 에러 케이스

| 케이스 | 처리 |
|---|---|
| 비이미지 파일 업로드 | 415 + 메시지 |
| 10MB 초과 | 413 + 메시지 |
| 스테이지당 10장 초과 | 409 + 메시지 |
| Blob 업로드 성공 → 클라이언트 저장 전 떠남 | orphan blob 발생 (운영 정기 정리 v2 — 현재는 허용) |
| 이미지 삭제했는데 stages 저장 안 함 | 클라이언트가 stages 저장 시 정합성 자동 회복 |
| 라이트박스에서 다음 이미지 없음 | 화살표 비활성화 |
