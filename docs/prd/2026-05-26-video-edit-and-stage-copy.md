# PRD: 관리자 영상 편집 + 스테이지 설명 전체 복사

- 작성일: 2026-05-26
- 작성자: Claude (요청자: ohdongko)
- 관련 파일: [components/AdminVideos.tsx](../../components/AdminVideos.tsx), [components/VideoPage.tsx](../../components/VideoPage.tsx)
- 범위: 클라이언트 UI (localStorage 기반, 서버 변경 없음)

---

## 1. 배경 / 문제

현재 관리자 페이지(`AdminVideos`)에서는 **이미 등록된 영상의 제목과 YouTube URL을 수정할 수 없다**. 레벨/스테이지/순서는 인라인 편집 가능하나, 제목·URL은 한 번 등록하면 삭제 후 재등록해야만 변경할 수 있어 운영상 비효율이 크다.

사용자 측 영상 시청 모달(`VideoPage`)의 학습 단계 아코디언에서, 강사가 단계 안에 입력한 **상세 설명(프롬프트, 가이드)을 한 번에 복사**할 수 있는 수단이 없다. 학습자는 마우스로 드래그해 선택·복사하고 있어 모바일·터치 환경에서 불편하다.

## 2. 목표 / 비목표

### 목표
- G1. 관리자 영상 목록에서 **영상 제목**과 **YouTube URL**을 인라인으로 수정 후 저장할 수 있다.
- G2. 시청 모달의 각 스테이지 아코디언이 열렸을 때, **토글 내부 가장 상단 맨 왼쪽**에 잘 보이는 "전체 복사" 버튼이 있고, 클릭 시 해당 스테이지의 `description` 전체를 클립보드로 복사한다.
- G3. 복사 성공/실패에 대한 시각적 피드백(임시 "복사됨" 라벨)을 제공한다.

### 비목표
- 서버 API/DB 스키마 변경 (현재 영상 데이터는 `localStorage` 저장)
- 영상 썸네일·설명문·레벨 등 다른 필드 편집 기능 추가 (제목/URL만)
- 여러 스테이지 일괄 복사, 영상 전체(제목+모든 단계) 복사
- 권한 분리 (현재 admin은 이미 단일 비번 모드)

## 3. 사용자 시나리오

### S1. 영상 정보 수정
1. 관리자 로그인 → 영상 관리 탭.
2. "등록된 영상 목록" 항목 우측의 **"편집"** 버튼 클릭.
3. 해당 행이 편집 모드로 전환 — 제목 input과 URL input 노출.
4. URL 변경 시 인식된 `videoId`를 즉시 미리보기 (현재 등록 폼과 동일 UX).
5. **저장** 클릭 → 변경 반영, 토스트 메시지. **취소** 클릭 → 변경 폐기.
6. 필수값(제목/URL) 비어있거나 URL이 유효하지 않으면 저장 버튼 비활성화.

### S2. 스테이지 설명 복사
1. 학습자가 영상 카드 클릭 → 시청 모달.
2. 학습 단계 아코디언에서 특정 단계 클릭 → 펼침.
3. 펼쳐진 영역 **가장 상단 맨 왼쪽**에 "📋 전체 복사" 버튼이 보임.
4. 클릭 시 `stage.description` 전체가 클립보드로 복사되고, 버튼 라벨이 1.5초간 "✓ 복사됨"으로 변함.
5. `description`이 비어있는 스테이지에서는 버튼이 표시되지 않거나 비활성화 (현재 description이 없으면 펼침 영역 자체가 렌더되지 않으므로 자연스럽게 미노출).

## 4. 기능 요구사항

### F1. 영상 인라인 편집 (AdminVideos)
- F1.1. 각 영상 행에 "편집" 버튼 추가. 위치: 기존 "스테이지" 버튼 좌측, "삭제" 버튼 우측 (목록 행 우측 그룹).
- F1.2. 편집 모드 진입 시 해당 행 전체가 편집 UI로 전환되며, 제목/URL input과 저장/취소 버튼이 노출된다. 다른 행과 동시 편집은 단순화를 위해 한 번에 한 행만 허용 (`editVideoId` state로 관리, 기존 `editStagesId`와 같은 패턴).
- F1.3. 저장 시 `extractVideoId(url)` 결과가 `null`이면 저장 거부(저장 버튼 비활성화 + 입력 아래 인식 불가 표시).
- F1.4. 저장하면 `persist()`로 즉시 반영. 성공 토스트: "영상 정보가 수정되었습니다."
- F1.5. 취소 시 변경 폐기, 편집 모드 종료.
- F1.6. 편집 중에는 동일 행의 순서 변경/레벨 변경/삭제/스테이지 토글 버튼을 비활성화 (의도치 않은 동시 조작 방지).

### F2. 스테이지 전체 복사 버튼 (VideoPage)
- F2.1. 펼쳐진 스테이지 콘텐츠 영역의 **첫 번째 자식**으로 복사 버튼을 배치 (description 텍스트 위, 좌측 정렬).
- F2.2. 버튼 라벨: 기본 "📋 전체 복사", 복사 직후 1.5초간 "✓ 복사됨" 으로 변경.
- F2.3. 클릭 시 `navigator.clipboard.writeText(stage.description)` 호출. `clipboard` API 미지원 또는 권한 거부 시 fallback으로 `document.execCommand('copy')` 사용 후, 그래도 실패하면 alert(`복사에 실패했습니다.`).
- F2.4. 동시에 여러 스테이지가 펼쳐질 수 없으므로 (`openStageIdx` 단일 인덱스), 복사 상태도 단일 boolean로 충분.
- F2.5. 버튼 스타일: 작은 secondary 버튼 (배경 `T.surface`, 보더 `T.border`, 호버 시 `T.primaryLight`). 모달 폭이 좁아도 항상 클릭 가능하도록 min-height 28px 확보.

## 5. UX/디자인

### F1 편집 행 레이아웃 (예시)
```
[↑↓] [순번] [레벨▼]  [제목 input ─────────][URL input ─────][videoId 미리보기]  [저장] [취소]
```
- 좁은 폭에서는 제목/URL이 두 줄로 wrap. (Tailwind `flex-wrap` 사용 가능)

### F2 펼침 영역 레이아웃 (변경 후)
```
┌──────────────────────────────────────────────┐
│ [📋 전체 복사]                                  │  ← 새로 추가, 좌측 정렬, padding-top:8
│                                              │
│ {stage.description}                          │
└──────────────────────────────────────────────┘
```
- 기존 `padding: '10px 14px 12px 46px'`에서 button row를 위한 padding-top 8px 추가, description은 줄바꿈 유지.

## 6. 데이터 / 상태 변경

- 신규 state (AdminVideos): `editVideoId: string | null`, 편집 중 임시 `editTitle/editUrl` (편집 모드 진입 시 현재 값으로 초기화).
- 신규 state (VideoPage): `copiedStageIdx: number | null` (1.5초 후 자동 해제).
- 스토리지 스키마 변경 없음 — 기존 `Video.title`, `Video.youtubeUrl` 그대로 사용.

## 7. 엣지 케이스

| 케이스 | 동작 |
|---|---|
| 편집 중 모달 전체에서 storage 이벤트 발생 (다른 탭 수정) | 현재 편집 폼은 그대로 유지하되, 저장 시 최신 목록 위에 머지. (이미 다른 곳도 동일 패턴이므로 별도 가드 없음) |
| URL 형식 부적합 (`youtu.be`, `shorts` 등) | `extractVideoId` 결과를 신뢰. 인식 불가 시 저장 버튼 비활성화. |
| 제목 공백만 입력 | 저장 거부. trim 후 빈 문자열이면 비활성. |
| stage.description이 빈 문자열 | 펼침 영역 미렌더 (현행 유지) → 복사 버튼도 자연히 미노출. |
| `navigator.clipboard` 미지원 (구형 브라우저 / 비-HTTPS) | execCommand fallback. 둘 다 실패 시 alert. |
| description에 이모지/멀티라인 | `whiteSpace: 'pre-wrap'`이므로 그대로 복사됨. |

## 8. 보안 / 권한

- Admin 비번은 이미 클라이언트 상수 (데모용)이므로 추가 권한 처리 없음.
- 복사 대상은 사용자에게 이미 노출된 텍스트 — 추가 노출 위험 없음.
