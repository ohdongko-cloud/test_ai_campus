# PRD: YouTube 영상 외부 유출 방어(옵션 A) + 가이드 아이콘/설명 개편 + Figma 추가

## 1. 배경

### A. 영상 외부 유출 위험
강의 영상 다수가 YouTube **일부공개(unlisted)** + 사이트 임베드 구조. 운영진 우려:
- 시청자가 YouTube 플레이어의 "YouTube에서 시청" / 좌하단 로고 클릭 → YouTube 페이지 진입 → 주소창에서 URL 복사 → 외부 유포 가능.
- 우클릭 메뉴 "동영상 URL 복사", 개발자도구 등 추가 경로 다수.

**기술적 한계 확인**: YouTube가 `modestbranding=1` 효과를 2023년 11월부터 무력화 → 플레이어 내부 외부 링크 완전 차단 불가능.

### B. 가이드 아이콘 / 설명 가독성
- 현재 가이드 카드 아이콘은 서비스명 앞 글자(CC, CO 등) — 인지율 낮음.
- 설명 톤이 기술 위주("Anthropic Sonnet 4.5 기반...") → AI 입문자에 어려움.
- 그래픽 디자인 그룹에 Figma 누락.

## 3. 목표 / 비목표

### 목표
- G1. 영상 시청 모달에서 **우클릭·단축키 차단**. YouTube **외부 링크 영역에 클릭 차단 오버레이**.
- G2. 영상 위에 **시청자 이메일·시각 워터마크** (반투명·이동) — 유출 시 추적용.
- G3. 영상 모달 하단에 **"본 영상은 사내 한정 자료입니다. 외부 공유 금지."** 안내.
- G4. 가이드 카드에 **실제 서비스 로고 아이콘** 표시 (Simple Icons / Google favicon fallback).
- G5. 그래픽 디자인 그룹에 **Figma** 추가 (3 아이템).
- G6. 모든 가이드 아이템(32개) 설명을 **초보자 친화** 톤으로 재작성.

### 비목표
- 영상 완전 비공개(Vimeo/Stream 이전) — 옵션 B로 분리.
- 가이드 아이콘을 관리자 페이지에서 수정 가능하게 변경.
- 사용자별 시청 기록·진도 추적.

## 4. 기능 요구사항

### F1. 영상 보호 (VideoPage.tsx)

#### F1.1. 우클릭/단축키 차단
- 영상 시청 모달 전체에 `onContextMenu={e => e.preventDefault()}`.
- 단축키 차단: Ctrl+S, Ctrl+U, Ctrl+Shift+I, Ctrl+Shift+J, F12, Ctrl+C 모달 내부에서 차단 (모든 차단 효과 제한적이지만 캐주얼 deter).
- iframe `sandbox` 속성에 `allow-scripts allow-same-origin allow-presentation` 만 허용 (다운로드 시도 차단).

#### F1.2. YouTube 외부 링크 오버레이
- 영상 컨테이너 우상단/좌하단에 **투명한 CSS 오버레이** 배치:
  - 우상단: 60×42px (모바일 80×40px) — "YouTube에서 시청" 버튼 위치
  - 좌하단: 90×30px — YouTube 로고 위치
- `z-index: 5`, `pointer-events: auto`, `cursor: not-allowed`.
- 클릭 시 노출 메시지: 토스트 "외부 사이트로 이동할 수 없습니다."
- iframe url 파라미터: `?rel=0&modestbranding=1&iv_load_policy=3&disablekb=1&playsinline=1`.

#### F1.3. 워터마크
- iframe 컨테이너에 `position: absolute`로 워터마크 div 4개 (모서리).
- 내용: `{user.email || '익명'} · {now}` (반투명 흰색 12px, 반투명 검정 그림자).
- 30초마다 위치 swap (좌상↔우하, 우상↔좌하) → 가리기 어려움.
- `pointer-events: none` — 클릭은 영상으로 통과 (사용자 인터랙션 방해 X).
- 사용자 이메일은 `/api/users/me`에서 가져옴 (로그인 안 한 경우 sessionId 마지막 8자).

#### F1.4. 외부 공유 금지 안내
- 모달 푸터(닫기 버튼 아래)에 빨간 작은 텍스트:
  - "🔒 본 영상은 이랜드리테일 사내 한정 자료입니다. 외부 공유·녹화·캡처를 금지합니다."

### F2. 가이드 카드 아이콘

#### F2.1. 아이콘 매핑
- `components/GuidePage.tsx`의 카드에 아이콘 영역 추가 (현재 텍스트 placeholder 자리).
- 우선순위:
  1. 화이트리스트 매핑 (`SERVICE_ICONS[name.toLowerCase()]` → URL)
  2. Simple Icons CDN: `https://cdn.simpleicons.org/{slug}/{color}` (일부 서비스)
  3. Google Favicon API: `https://www.google.com/s2/favicons?domain={domain}&sz=128`
- 정사각형 강제 (`object-fit: contain`, 흰 배경, 라운드 코너).

#### F2.2. 아이콘 라이브러리 (lib/service-icons.ts)
- 32개 서비스에 대해 (서비스명 lower → 아이콘 URL) 매핑.
- 가능하면 Simple Icons SVG 사용 (브랜드 색상).
- 미지원 서비스는 favicon API.

## 7. 엣지 케이스

| 케이스 | 동작 |
|---|---|
| 비로그인 사용자 영상 시청 | 워터마크에 sessionId 마지막 8자 표시 (`anon · abc12345`) |
| 모바일 화면 우상단 영역 작음 | 미디어 쿼리로 오버레이 크기 조정 (60% 비율 유지) |
| 워터마크가 영상 콘텐츠 가림 | 모서리 위치만 사용 + 반투명 (opacity 0.5) |
| 서비스 아이콘 로드 실패 | onError 시 텍스트 fallback (이전 CC/CO 형태) |
| 신규 서비스 추가 시 아이콘 없음 | 매핑 누락 → favicon API로 자동 처리 |
| Figma 그룹·그래픽 카드 늘어남 | 그리드 자동 줄바꿈 (기존 동작 유지) |

## 8. 보안

- 워터마크 이메일은 클라이언트 표시일 뿐 — 실제 보안은 사회적 억제력에 의존.
- 모든 차단은 deterministic 우회 가능 (개발자도구) → 옵션 B로 격상 가능성 PRD 별도 추적.
- iframe sandbox는 다운로드 일부 차단하지만 fullscreen 제한될 수 있음 → 사용성과 trade-off.
