# PRD: 비밀번호 보기 토글 (눈 아이콘)

- 작성일: 2026-05-26
- 작성자: Claude (요청자: ohdongko)
- 범위: 클라이언트 UI (WelcomePopup.tsx)

---

## 1. 배경 / 문제

로그인 화면에서 비밀번호는 `*` 처리되어 사용자가 본인이 무엇을 입력했는지 확인 불가. 모바일 가상 키보드 + 4~12자 간편 비번 조합에서 오타 발생 빈도가 높음. 회원가입 시 비번 2회 입력으로 일부 보완되지만 **로그인 화면에서는 1회 입력이라 오타 시 그냥 401**.

## 2. 목표 / 비목표

### 목표
- G1. 로그인 화면 비번 input 우측에 **눈 아이콘 토글** 버튼 추가. 누르면 평문 표시, 다시 누르면 마스킹.
- G2. 회원가입의 비번 / 비번 확인 input에도 동일 적용 (일관성).
- G3. 토글 상태는 input 단위로 독립 (한 input 보여도 다른 input은 마스킹 유지).

### 비목표
- 비번 강도 미터.
- 자동완성 비활성/활성.
- 비번 붙여넣기 차단.

## 3. UX

```
간편 비밀번호 *
[●●●●●●●●●        ][👁]
```
- 눈 아이콘은 보이는 상태일 땐 "/" 그어진 눈 (`eye-off`), 가려진 상태일 땐 일반 눈 (`eye`).
- 입력 영역 우측 12px 안에 배치, 입력 영역 폭에 영향 없음.

## 4. 기능

- WelcomePopup.tsx에서 3개 비번 input (`pw`, `pwConfirm`, `loginPw`)을 헬퍼 `PasswordField`로 래핑.
- 각각 독립된 `useState<boolean>` 으로 visible 상태 관리.

## 5. 엣지 케이스

- 모바일 가상 키보드 호환: 표시 모드는 `type="text"`, 마스킹은 `type="password"` — 표준 패턴.
- 자동완성: 브라우저가 자체 채우기 시 토글 상태와 무관하게 입력값 그대로 동작.

## 6. 성공 기준

- [ ] 로그인 입력란 우측 눈 아이콘 표시.
- [ ] 클릭 시 평문 ↔ 마스킹 전환.
- [ ] 회원가입의 두 비번 input도 동일 동작.
- [ ] 빌드 통과.
