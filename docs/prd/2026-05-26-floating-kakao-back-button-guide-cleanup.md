# PRD: 플로팅 카톡 버튼 + 뒤로가기 동작 + 가이드 헤더 정리

- 작성일: 2026-05-26
- 작성자: Claude (요청자: <오너>)
- 범위: 카카오톡 FAB(Floating Action Button) 추가 / 탭 hash 라우팅 / 가이드 stats 제거

---

## 1. 배경 / 문제

### A. 오픈 채팅방 진입 동선 부족
- 오픈 채팅방은 사이트 전체에서 가장 자주 권장되는 액션. 현재는 홈 화면 카드 1개만 진입로 — 다른 탭에 있을 때는 진입 불가.
- 카드 형태는 시각적 우선순위가 낮아 놓치기 쉬움.

### B. 브라우저 뒤로가기 작동 불가
- React state 기반 SPA로 탭 전환 시 URL이 바뀌지 않음 → 브라우저 history에 기록 안 됨 → 뒤로가기 무력화.
- 모바일 사용자 특히 불편 (안드로이드 뒤로가기 버튼 = 사이트 종료).

### C. 가이드 페이지 헤더 stats 노출 불필요
- 헤더에 "10 카테고리 / 31 필수·추천 서비스 / 19 사내 추천 / 2026.05 마지막 업데이트" 노출.
- 운영 입장에서 의미 없는 데이터 + 카테고리 수 자동 갱신 어려움.

## 2. 목표 / 비목표

### 목표
- G1. **카카오톡 모양 FAB** — 모든 탭에서 우측 하단 고정. 스크롤해도 자리 유지. 클릭 시 오픈 채팅방 새 탭.
- G2. **기존 홈 'AI 오픈채팅방 참여' 카드 제거** (FAB로 대체).
- G3. **탭 클릭 시 URL hash 동기화** + **뒤로가기로 직전 탭 복귀**.
- G4. **가이드 헤더 stats 블록 제거**.

### 비목표
- 카톡 봇/딥링크 연동.
- FAB 드래그 가능 위치.
- 가이드 헤더 다른 정보 추가.
- 경로 라우팅(/videos, /board) 전환.

## 3. 기능 요구사항

### F1. 카카오톡 FAB
- 위치: `position: fixed`, `right: 20px`, `bottom: 20px` (모바일: `right: 16px`, `bottom: 16px`).
- 크기: 56×56 (모바일 52×52), 정사각형 + 큰 모서리 라운드(`border-radius: 16`).
- 배경: 카카오 옐로 `#FEE500`.
- 아이콘: 카카오톡 말풍선 SVG (검정 #181600, 32px).
- 호버: 살짝 확대(`scale(1.05)`) + 더 진한 그림자.
- z-index: 999 (모달보다 낮게, 일반 컨텐츠보다 높게. 시청 모달 z-index 1000과 충돌 없음).
- 표시 영역: 어드민 모드 제외 모든 탭. 환영 팝업·기타 모달 노출 중에는 z-index 가려져도 무방.
- 클릭 시:
  - `addClickLog('오픈채팅방 FAB')` 통계 집계.
  - `chatroom_url` 설정값을 `/api/settings`에서 가져와 새 탭 오픈.
  - URL 미설정 시 카카오 공식 도메인 fallback.
- 접근성: `aria-label="오픈 채팅방 열기"`, `title="AI 오픈 채팅방"`.

### F2. 기존 홈 카드 제거
- `components/MainPage.tsx`의 "AI 오픈채팅방 참여" `ActionCard` 블록 삭제.
- 같은 행의 나머지 2 카드(멘토링 / Q&A) 그리드 `ac-grid-2` 로 자동 적응.

### F3. Hash 라우팅 (뒤로가기 동작)
- 탭 정의:
  - `home` → `` (hash 없음 = home 기본)
  - `videos` → `#videos`
  - `meeting` → `#meeting`
  - `board` → `#board`
  - `share` → `#share`
  - `guide` → `#guide`
- 탭 클릭(navigateTo) 시 `history.pushState(null, '', '#' + tab)` (단, 같은 탭이면 skip).
- 초기 로드 시 hash 읽어 activeTab 설정.
- `window.addEventListener('popstate', ...)` 로 뒤로가기/앞으로가기 감지 → activeTab 동기화.
- 어드민 모드 진입/이탈은 별도(URL 영향 안 줌).

### F4. 가이드 헤더 stats 제거
- `components/GuidePage.tsx`의 hero stats 블록(10·31·19·2026.05) 삭제.
- 우측 카테고리 chip 목록은 유지(빠른 점프 기능).

## 6. 엣지 케이스

| 케이스 | 동작 |
|---|---|
| 모달 열린 상태에서 FAB 클릭 | z-index 1000이 더 위에 있어 FAB 가려짐. 의도된 동작. |
| 어드민 페이지 진입 | FAB 미표시 (관리자 페이지 컴포넌트 별도). |
| chatroom_url 미설정 | fallback `https://open.kakao.com/`. |
| 사용자가 hash만 변경 (`#unknown`) | 알 수 없는 값이면 home으로 fallback. |
| 외부 링크로 들어왔다가 뒤로가기 | 사이트 진입 직전 페이지로 정상 이동(브라우저 기본 동작). |
| 모바일 가상 키보드 띄운 상태에서 FAB | 키보드에 가려질 수 있음 — 허용. |
| FAB 영역에 다른 요소(채팅 모달) 겹침 | 모달은 z-index 더 높게 유지. |
