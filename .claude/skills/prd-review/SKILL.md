# /prd-review — PRD 리뷰 (범용 품질 게이트)

PRD가 구현하기에 충분한지 루브릭으로 점검한다.

## 0. 규약 탐지
- `CLAUDE.md`에 보안정책·특화 체크가 있으면 4·5·6번 항목을 **그 정책에 매핑**. 없으면 일반 기준 적용.
- PRD 경로: 인자 → 없으면 PRD 폴더(docs/prd 등) 최근 문서.

## 절차
1. 대상 PRD를 읽는다.
2. 아래 8항목을 **✅ 통과 / ⚠️ 보강 / ❌ 미흡**으로 판정.
3. ⚠️·❌마다 **구체적 수정 문장**을 제시(추상적 지적 금지).
4. 원하면 PRD를 직접 보강 편집한다.

## 루브릭 8항목
| # | 항목 | 합격 기준 |
|---|---|---|
| 1 | 성공기준 검증가능 | 동작으로 확인되는 체크박스 + "빌드/테스트 통과" 포함 |
| 2 | 비목표 명시 | 스코프 제외 경계가 적혀 크리프를 막는가 |
| 3 | 엣지 케이스 실질 | 실패·경계·동시성 입력이 표로 정리됐는가 |
| 4 | 보안·PII 영향 | 인증·인가·레이트리밋·PII·에러 누출 (CLAUDE.md 정책 있으면 매핑) |
| 5 | DB 마이그레이션 | 신규 컬럼/테이블·멱등·하위호환·기본값이 명시됐는가 |
| 6 | 플랫폼·호환 영향 | 모바일/멀티플랫폼·버전·하위호환이 고려됐는가 |
| 7 | 의존성·사전조건 | env·외부서비스·선행 PRD가 적혔는가 |
| 8 | 미해결 질문 | 결정 못 한 항목을 드러냈는가 |

> 4·5·6은 해당 기능이 그 영역을 건드리지 않으면 "해당 없음(N/A)" 표기 가능. 단 판단 근거 한 줄.

## 출력 형식
```
[prd-review] <경로>
1. 성공기준 검증가능 ✅
2. 비목표 ⚠️ — "<무엇>"이 비목표에 없음 → "<수정 문장>"
...
종합: ⚠️ 보강 후 구현 권장 (❌ n, ⚠️ m)
다음: 보강 → 구현 → /prd-commit
```
- ❌ 있으면 "구현 전 보강 필요", 전부 ✅/N/A면 "구현 착수 가능".
