# /prd-flow — 기획→배포 오케스트레이터

5개 스킬을 **정답 루트**로 잇는다. 게이트 누락·순서 사고를 막는 게 목적. 각 단계 사이 사용자 확인을 받는다.

## 순서
1. **PRD 작성** — `/prd-new` (이미 PRD가 있으면 건너뜀)
2. **리뷰 = 게이트①** — `/prd-review`. ❌ 있으면 PRD 보강 후 1~2 반복.
3. **구현** — F1·F2… 요구사항을 코드로 구현(사람 + AI). 진행 중 `CLAUDE.md` 특화 체크 9 준수.
4. **커밋** — `/prd-commit` (PRD 단위 원자적 커밋 + CHANGELOG).
5. **보안 = 게이트②** — `/security-gate`. 🚫 있으면 수정 후 4~5 반복.
6. **배포** — `/ship` (빌드 게이트 → 푸시 확인 → 배포 검증).

## 흐름 제어
- 게이트(②리뷰 / ⑤보안)에서 막히면 → **구현/PRD로 되돌아가 수정 후 재진입**.
- 각 단계 시작 전 무엇을 할지 한 줄로 알리고, 완료 후 다음 단계로 갈지 확인.
- 단계는 건너뛸 수 있다(예: PRD 이미 있음 → 1 생략). 단, 두 게이트(②·⑤)는 건너뛰지 않는다.

## 입력
- 기능 한 줄 설명(인자). 없으면 현재 변경/`docs/prd` 최근 문서에서 시작점을 추론한다.

## 참고
- 각 단계 상세: `/prd-new` · `/prd-review` · `/prd-commit` · `/security-gate` · `/ship`
- 공통 기준: `CLAUDE.md`
