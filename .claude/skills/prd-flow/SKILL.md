# /prd-flow — 기획→배포 오케스트레이터 (범용)

작성·리뷰·구현·마이그레이션·커밋·보안·푸시를 **정답 루트**로 잇는다. 각 단계 사이 사용자 확인을 받는다.

## 0. 규약 탐지 (먼저)
- `CLAUDE.md`/PRD 폴더/`package.json`/마이그레이션 도구/배포 타깃을 한 번 파악해 하위 스킬들이 일관되게 적응하도록 한다. 없으면 각 스킬의 기본 동작.

## 순서
1. **PRD 작성** — `/prd-new` (이미 PRD가 있으면 건너뜀)
2. **리뷰 = 게이트①** — `/prd-review`. ❌면 PRD 보강 후 1~2 반복.
3. **구현** — 요구사항(F1·F2…)을 작업 단위로 구현. (CLAUDE.md가 있으면 그 규칙 준수)
4. **마이그레이션** — DB 스키마가 바뀌면 `/db-migrate`(멱등·하위호환). 없으면 건너뜀.
5. **커밋** — `/prd-commit` (작업 단위 원자적 커밋 + CHANGELOG).
6. **보안 = 게이트②** — `/security-gate`. 🚫면 수정 후 4~6 반복.
7. **배포** — `/ship` (빌드 게이트 → 푸시 확인 → 배포 검증 → 운영 마이그레이션 안내).

## 흐름 제어
- 게이트(②리뷰 / ⑥보안)에서 막히면 → **구현/PRD로 되돌아가 수정 후 재진입**.
- 각 단계 시작 전 무엇을 할지 한 줄로 알리고, 완료 후 다음 단계로 갈지 확인.
- 단계 건너뛰기 허용(예: PRD 이미 있음 → 1 생략, 스키마 변경 없음 → 4 생략). 단, 두 게이트(②·⑥)는 건너뛰지 않는다.

## 입력
- 기능 한 줄 설명(인자). 없으면 현재 변경/최근 PRD에서 시작점을 추론한다.

## 참고
- 각 단계: `/prd-new` · `/prd-review` · `/db-migrate` · `/prd-commit` · `/security-gate` · `/ship`
- 공통 기준: repo의 `CLAUDE.md`(있으면).
