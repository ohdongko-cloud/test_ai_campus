# /prd-new — PRD 작성

이 프로젝트의 **8섹션 고정 템플릿**으로 PRD 초안을 만든다. 결과는 `docs/prd/`에 저장하고 리빙독스에 등록한다.

## 절차

### 1. 범위 좁히기
- 기능 한 줄 설명을 받는다(인자 또는 질문).
- 모호하면 **핵심 2~3개만** 질문한다: 대상 사용자/화면? 포함·제외 경계? 제약(보안·DB·외부연동)?
- 충분하면 질문 없이 진행한다.

### 2. 기존 톤 학습
- `docs/prd/`의 최근 문서 1~2개를 읽어 구조·문체·표기를 맞춘다(예: `2026-05-26-add-logout-button.md`).

### 3. 8섹션 작성
1. **배경 / 문제** — 왜 필요한가
2. **목표 / 비목표** — G1·G2…(달성 목표) / 비목표(스코프 제외)
3. **사용자 시나리오** — S1·S2… 단계별 흐름
4. **기능 요구사항** — F1·F2… 구체·검증 가능하게
5. **UX / 디자인** — 배치·상태·문구(필요 시 ASCII 스케치)
6. **엣지 케이스** — 표(케이스 | 동작)
7. **성공 기준** — 체크박스(클릭/동작으로 확인 가능 + "빌드 통과")
8. **미해결 질문** — 결정 못 한 것

### 4. 보안·영향 섹션 자동 추가 (이 프로젝트 특화)
기능이 다음을 건드리면 해당 항목을 PRD에 **명시**한다(이후 `/prd-review`·`/security-gate`가 검사):
- **인증/권한** → §6 매핑(세션·master/admin/user·레이트리밋·enumeration)
- **PII/개인정보** → 응답 제외·로그 금지·캐시 정책
- **DB** → 신규 컬럼/테이블 + 멱등 마이그레이션(`/api/admin/migrate`) + 하위호환
- **모바일/안드로이드** → `app/m/*` 라우트, Capacitor `versionCode`
- **env** → 새 변수 + `.env.local.example`·`.gitleaks.toml` 동기화
(기준: `CLAUDE.md` §6 / 특화 체크 9)

### 5. 저장
- 파일명: `docs/prd/<오늘날짜>-<slug>.md` — 날짜는 `date +%F`, slug는 영문 케밥(예: `video-attachments`).
- 헤더: 작성일 / 작성자: Claude (요청자: <오너>) / 범위.

### 6. 등록 (리빙독스)
- `docs/prd/CHANGELOG.md` §2 "작성된 PRD 목록"에 행 추가.
- 아직 안 할 작업이면 `docs/prd/BACKLOG.md`에 우선순위(P0~P3)로 등록.

### 7. 마무리
- 저장 경로를 보여주고 **`/prd-review`로 품질 점검**을 권한다.
