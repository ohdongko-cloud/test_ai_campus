# /prd-new — PRD 작성 (범용)

새 기능의 PRD 초안을 만든다. **프로젝트 규약이 있으면 따르고, 없으면 기본 템플릿**으로 동작한다.

## 0. 프로젝트 규약 탐지 (먼저)
- `CLAUDE.md`(또는 `AGENTS.md`, `.cursorrules`) → PRD 템플릿·작성 규칙의 기준.
- PRD 폴더: `docs/prd/` → `docs/specs/` → `docs/rfcs/` → `prd/` 순으로 탐색. 있으면 그 위치·파일명 규칙·기존 문서 톤을 따른다.
- `CHANGELOG.md`/PRD 인덱스가 있으면 등록 대상으로 기억.
- 위가 없으면 → 아래 **기본 8섹션 템플릿** + 저장 위치 `docs/prd/`(없으면 생성, 사용자 확인).

## 1. 범위 좁히기
- 기능 한 줄 설명을 받는다(인자/질문). 모호하면 핵심 2~3개만 질문: 대상·화면? 포함/제외 경계? 제약(보안·DB·외부연동)?

## 2. 기존 톤 학습
- 탐지된 PRD 폴더의 최근 문서 1~2개를 읽어 구조·문체·표기를 맞춘다.

## 3. 작성 — 8섹션 기본 템플릿
1. **배경/문제** — 왜 필요한가
2. **목표/비목표** — G1·G2…(목표) / 비목표(스코프 제외)
3. **사용자 시나리오** — S1·S2… 단계별 흐름
4. **기능 요구사항** — F1·F2… 구체·검증 가능
5. **UX/디자인** — 배치·상태·문구
6. **엣지 케이스** — 표(케이스 | 동작)
7. **성공 기준** — 체크박스(동작으로 확인 + "빌드/테스트 통과")
8. **미해결 질문**

## 4. 영향 섹션 (해당 시 자동 추가)
기능이 다음을 건드리면 PRD에 명시한다(이후 `/prd-review`·`/db-migrate`·`/security-gate`가 검사):
- **인증/권한·PII** → 보안 영향(CLAUDE.md에 정책 있으면 그 항목에 매핑, 없으면 일반: 인증·인가·레이트리밋·PII 응답 제외·에러 누출).
- **DB 스키마** → 신규 컬럼/테이블·**멱등·하위호환·기본값**·적용 방법.
- **모바일/플랫폼·env** → 영향 라우트·새 환경변수·예시파일 동기화.

## 5. 저장 & 등록
- `<PRD폴더>/<YYYY-MM-DD>-<slug>.md` — 날짜 `date +%F`, slug 영문 케밥.
- 헤더: 작성일 / 작성자 / 범위.
- `CHANGELOG`/PRD 인덱스가 있으면 한 줄 등록.

## 6. 마무리
- 저장 경로를 보여주고 **`/prd-review`**(품질 점검)를 권한다.
